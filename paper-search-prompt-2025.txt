# 논문 검색 프롬프트 (2025년 12월 기준)

## 연구 배경 및 목표

저는 **소형 언어모델(SLM, 3B~8B)의 추론 능력 향상**을 연구하고 있습니다.

### 핵심 연구 주제
1. **Structured Rationale Distillation**: LLM(70B+)이 생성한 구조화된 중간표현(Concept Graph, Reasoning Process)을 SLM에 증류
2. **CoT vs Structured Representation**: 서술형 Chain-of-Thought와 구조화된 표현(JSON Triple, Graph) 간 학습 효율성 비교
3. **Test-time Scaling for SLMs**: SLM에서 Self-consistency, Best-of-N, Verifier 등을 통한 추론 성능 향상
4. **Model Merging & Alignment**: 서로 다른 추론 스타일(CoT/CG)로 학습된 Adapter를 병합하고 DPO로 정렬

### 현재 달성 성과
- LLaMA-8B에서 CG 증류로 GSM8k-kor 성능 +7.0%p 향상 (49.2% → 56.2%)
- LLaMA-3B에서 RP 증류로 다교과 문제 +16.3%p 향상

### 연구 질문
- RQ1: SLM에 대해 서술형 CoT와 구조화 CG 중 어떤 중간표현이 더 데이터 효율적인가?
- RQ2: SLM에서 가장 비용 효율적인 Test-time scaling 방식은 무엇인가?
- RQ3: 서로 다른 추론 스타일(CoT/CG)을 병합하면 능력이 합산되는가?

## 검색 요청 1: Structured Rationale Distillation

**2023년 ~ 2025년**에 발표된 논문 중, 다음 주제와 관련된 연구를 찾아주세요:

### 키워드
- Knowledge Distillation for LLMs/SLMs
- Rationale Distillation / Chain-of-Thought Distillation
- Structured Output Distillation (JSON, Graph, Triple)
- Multi-task Learning with Rationales
- Teacher-Student Framework for Reasoning

### 관심 포인트
- LLM → SLM 증류 시 **구조화된 중간표현**(그래프, 트리플, JSON)을 활용하는 방법
- CoT/Rationale을 **auxiliary loss**로 사용하는 multi-task 학습
- **소형 모델(≤10B)**에서의 수학/추론 성능 향상 사례

## 검색 요청 2: Test-time Scaling & Inference Optimization

**2023년 ~ 2025년**에 발표된 논문 중, 다음 주제와 관련된 연구를 찾아주세요:

### 키워드
- Test-time Compute Scaling
- Self-Consistency / Majority Voting
- Best-of-N Sampling / Reranking
- Verifier / Reward Model for Reasoning
- Process Reward Model (PRM) vs Outcome Reward Model (ORM)
- Inference-time Scaling Laws

### 관심 포인트
- **SLM에서의 Test-time scaling** 효과 (LLM 대비 효율성)
- Verifier/Reward Model을 활용한 **후보 선택** 전략
- "SLM × N회 추론"이 "LLM × 1회 추론"을 능가하는 **break-even point**

**2023년 ~ 2025년**에 발표된 논문 중, 다음 주제와 관련된 연구를 찾아주세요:

- LoRA Merging / Adapter Composition
- Multi-task LoRA / LoRA Hub
- Mixture of Experts (MoE) for Reasoning
- DPO / Preference Optimization after Merging

### 관심 포인트
- 서로 다른 태스크로 학습된 **Adapter를 병합**하는 기법
- 병합 후 **성능 간섭(Interference)** 문제 해결 방법
- 병합된 모델의 **출력 스타일 정렬** (DPO, RLHF)
