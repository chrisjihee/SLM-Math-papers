## 2. One-line Summary

RASC는 Self-Consistency에서 단순 answer majority만 보지 않고, 각 reasoning-answer sample의 reasoning path quality와 answer consistency를 함께 점수화하여 early stopping, weighted majority voting, faithful rationale selection을 수행하는 inference-time SC 효율화 방법이다.

## 3. 핵심 문제 설정

- 기존 Self-Consistency는 sample 수를 늘리면 성능이 좋아질 수 있지만, 고정된 large sample budget 때문에 over-sampling과 높은 inference cost가 발생한다.
- 기존 `AC`, `ESC` 같은 효율화 방법은 주로 final answer consistency만 보고 stop 여부를 결정한다.
- RASC는 여기에 reasoning path quality variation을 함께 반영해야 한다고 본다.
- 따라서 핵심 문제는 다음 두 가지다.
  - 언제 sampling을 멈출 것인가
  - 어떤 sample / rationale을 더 신뢰할 것인가
- 중요한 점은 이 논문의 선택 단위가 `path family`가 아니라 **homogeneous CoT-style sampling 과정에서 나온 sample / rationale / answer**라는 점이다.

## 4. 핵심 방법

- 각 generated sample을 `(reasoning path, answer)` pair로 본다.
- 이 pair에서 두 종류의 feature를 추출한다.
  - answer-level features
  - reasoning-level features
- 이 feature를 lightweight scorer, 주로 Logistic Regression에 넣어 `sufficiency score`를 계산한다.
- 추론 과정:
  - CoT sample 반복 생성
  - sufficiency score 계산
  - threshold `T` 이상인 sample을 high-quality buffer에 저장
  - buffer 크기 `N`에 도달하면 stop
  - sufficiency score를 weight로 사용해 weighted majority voting 수행
  - selected answer를 지지하는 rationale 중 최고 score rationale을 선택
- 따라서 이 방법은 엄밀한 PRM은 아니지만, **sample-level lightweight quality estimator**로 동작한다.

## 5. SLM-Math 관점의 재해석

- RASC는 현재 연구와 매우 직접적으로 연결된다.
- 공통점:
  - 제한된 TTC 안에서 sampling을 언제 멈출지 다룬다.
  - reasoning quality를 early signal로 활용한다.
  - lightweight scorer로 비용을 줄인다.
- 차이점:
  - RASC는 homogeneous CoT-style sample acquisition 안에서 sample quality를 고른다.
  - 내 연구는 `direct`, `CoT`, `prompt-diverse CoT`, `CG`, `PoT`, `structured rationale`, `verifier-guided path` 같은 **heterogeneous reasoning path family 또는 macro strategy**를 고르는 방향이어야 한다.
- 한 문장으로 정리하면:
  - RASC는 "같은 sampling process에서 몇 개 더 뽑을지, 어떤 sample을 믿을지"를 다룬다.
  - 내 연구는 "현재 pool state를 보고 다음에 어떤 종류의 reasoning path를 더 획득할지"를 다뤄야 한다.

## 6. 우리 연구에 대한 novelty risk

- `SC efficiency` 자체는 이미 새롭지 않다.
- `reasoning-aware stopping`도 이미 새롭지 않다.
- `weighted majority voting`도 이미 있다.
- `faithful rationale selection`도 이미 있다.
- `lightweight classifier로 TTC를 줄인다`는 주장도 이미 존재한다.
- 특히 위험한 claim:
  - "우리는 Self-Consistency에서 reasoning path quality를 보고 sampling을 효율화한다."
- 안전한 방향:
  - `sample-level homogeneous SC efficiency`가 아니라
  - `state-conditioned heterogeneous path-family acquisition and macro strategy allocation`

## 8. 우리가 하면 안 되는 주장

- Self-Consistency의 sample 수를 동적으로 줄이는 것이 새롭다는 주장
- final answer consistency만 보지 않고 rationale quality를 본다는 주장
- reasoning path quality 기반 early stopping의 최초성 주장
- weighted majority voting이 새롭다는 주장
- faithful rationale selection이 새롭다는 주장
- lightweight classifier 기반 TTC control이 새롭다는 주장
- `SC efficiency` 개선만으로 top-tier novelty가 충분하다는 주장

## 9. baseline / ablation 반영 아이디어

- RASC를 `P0 baseline`으로 명시하기
- `SC / AC / ESC / RASC` baseline을 vote bank simulator에서 재현하기
- `RASC-style feature set`을 state representation baseline으로 구현하기
- answer-level only / reasoning-level only / combined feature ablation 추가하기
- `# Gen`, `Acc`, `Gain/Sample`, `Avg Budget Used`를 공통 metric으로 채택하기
- inference overhead와 scoring/routing overhead를 분리 측정하기
- prompt-diverse CoT baseline에서 RASC류 stopping이 얼마나 통하는지 비교하기

## 10. Related Work에 넣을 문장 초안

RASC는 Self-Consistency의 inference-time 비용 문제를 다루는 대표적인 연구로, 기존 SC가 final answer majority에 의존하고 fixed sample 수를 사용하는 한계를 지적한다. 이 방법은 각 generated reasoning-answer pair에서 answer-level consistency와 reasoning-level quality feature를 추출하고, sufficiency score를 계산하여 early stopping과 weighted majority voting, faithful rationale selection을 수행한다. 그 결과 여러 reasoning benchmark에서 SC 수준의 accuracy를 유지하면서 sample 수와 inference time을 크게 줄였다.

그러나 RASC는 기본적으로 homogeneous CoT-style sampling 과정에서 생성된 개별 sample의 quality를 평가하고, 충분한 high-quality sample이 모이면 멈추는 방식이다. 즉, 선택 단위는 reasoning path family나 macro strategy가 아니라 개별 reasoning-answer sample이다. 반면 본 연구는 제한된 test-time compute 하에서 `direct`, `CoT`, `prompt-diverse CoT`, `structured rationale`, `PoT`, `verifier-guided path` 등 heterogeneous reasoning path family를 어떻게 구성하고, 현재 pool state에 따라 어떤 path family 또는 macro sampling strategy를 추가로 선택하며, 언제 멈출지를 다룬다. 따라서 본 연구는 RASC의 reasoning-aware early stopping 흐름을 중요한 선행연구로 수용하되, 문제를 sample-level SC efficiency에서 state-conditioned heterogeneous path-pool construction으로 확장한다.
