## 2. One-line Summary

STaR는 소수의 rationale 예시와 rationale 없는 대규모 데이터를 이용해, 모델이 스스로 rationale을 생성→정답인 것만 남겨 fine-tune하고, 틀린 문제는 정답을 hint로 준 rationalization으로 보강하는 과정을 반복하여 reasoning 능력을 **자기 부트스트랩**하는 self-taught reasoner 논문이다.

## 3. 핵심 문제 설정

- 문제: rationale 생성을 유도하려면 보통 대규모 rationale dataset 구축(비쌈) 또는 few-shot inference(정확도 희생)가 필요하다.
- 해법: **bootstrap loop** — (1) few-shot로 rationale+answer 생성, (2) 정답을 낸 rationale만 학습데이터로 채택해 fine-tune, (3) 틀린 문제는 **rationalization**(정답을 hint로 주고 역방향 rationale 생성)으로 보강, (4) 반복하며 점점 더 복잡한 reasoning 습득.
- 내 연구와의 연결: STaR는 **rationale bootstrapping / self-generated rationale로 self-improvement를 선점**했다. 우리 연구는 **training-time bootstrapping이 아니라 fixed backbone 위 test-time allocation**으로 구분된다.

## 4. 핵심 방법

- generation: few-shot CoT로 (rationale, answer) 생성.
- filtering: **정답 일치**한 rationale만 채택(self-filtering, outcome supervision).
- rationalization: 틀린 문제에 정답을 hint로 제공 → 역으로 rationale 생성 → 정답 유도분 채택.
- training: 채택된 rationale로 fine-tune, 다시 generation→filter→finetune **반복**.
- 학습: training-heavy(self-bootstrapping). test-time 제어/search/verifier 없음.
- selection unit: **rationale(self-generated, correctness-filtered)**. heterogeneous path-family 선택/budget allocation/STOP은 아님.
- 실험: base **GPT-J(6B)**, **GSM8K·CommonsenseQA·arithmetic**. few-shot/direct finetune 대비 향상.

## 5. SLM-Math 관점의 재해석

- STaR는 직접 baseline 경쟁자라기보다 **rationale-bootstrapping / self-improvement의 대표 reference이자 boundary**다.
- 공통점: rationale의 가치, 정답 기반 self-filtering(우리 voting/verification 직관과 닿음).
- 차이점:
  - STaR는 **training-time self-improvement(weight update 반복)**. 우리는 **fixed/given backbone 위 inference-time path-pool 구성·STOP**.
  - STaR의 correctness filtering은 학습 데이터 선별용. 우리는 inference-time에 voting/verifier로 최종 답 결정.
- 연결: STaR식 correctness-filtered self-generated rationale은 **(과거) distillation/bootstrapping 축**의 일부로 보존하되, 현재 기여는 그 위의 test-time orchestration임을 명문화. rStar-Math의 self-evolution과도 계보가 닿음(STaR가 그 원형 중 하나).

## 6. 우리 연구에 대한 novelty risk

- **rationale bootstrapping / self-generated rationale training / self-improvement via rationales**를 이미 선점 → 이 자체를 우리 novelty로 쓰면 충돌.
- "정답 기반으로 rationale을 self-filter해 학습하면 좋아진다"는 관찰도 선점.
- 위험 framing: `self-generated rationale training이 우리 기여`, `bootstrapping으로 SLM reasoning 향상`.
- 단, STaR는 training-time(미시 학습 루프)이라 우리 inference-time allocation과 결이 다름.

## 7. 우리가 빌릴 수 있는 것

- framing: rationale self-generation + correctness filtering, easy-to-hard 점증, data efficiency.
- baseline: STaR로 bootstrap한 backbone을 우리 orchestration의 **fixed backbone 후보**로(경쟁 아님).
- metric: accuracy + bootstrap iteration별 향상(training-time gain) — inference-time gain과 분해.
- ablation: rationalization on/off, filtering 기준, iteration sweep.
- terminology: `self-taught reasoner`, `bootstrapping`, `rationalization`, `correctness filtering`, `self-generated rationale`.

## 8. 우리가 하면 안 되는 주장

- `rationale bootstrapping / self-taught reasoning을 우리가 처음 제안한다`.
- `self-generated rationale training이 우리 기여다`.
- `정답 기반 self-filtering으로 reasoning을 self-improve하는 것을 우리가 처음 보였다`.
- 이 논문 대비 raw accuracy SOTA 경쟁, training-time gain을 inference-time 기여로 혼동.

## 9. baseline / ablation 반영 아이디어

- STaR/bootstrapping은 backbone 확보 수단으로만 두고(경쟁 아님), 비교 중심은 fixed backbone 위 test-time path-pool.
- correctness-filtered self-generation을 우리 inference-time voting/verification과 **목적 분리**(학습 데이터 선별 vs 최종 답 결정).
- training-time gain(bootstrap)과 inference-time gain(orchestration)을 명확히 분해.

## 10. Related Work에 넣을 문장 초안

STaR(Zelikman et al., NeurIPS 2022)는 소수의 rationale 예시와 rationale 없는 대규모 데이터로부터, 모델이 스스로 생성한 chain-of-thought rationale 중 정답을 낸 것만 채택해 fine-tune하고 틀린 문제는 정답을 hint로 준 rationalization으로 보강하는 과정을 반복하여 reasoning 능력을 자기 부트스트랩한다. 이는 self-generated rationale 기반 self-improvement의 대표적 원형이다.

본 연구는 rationale bootstrapping이나 self-generated rationale training 자체를 새 기여로 두지 않는다. STaR가 training-time에 weight를 반복 갱신하는 self-improvement인 반면, 우리는 fixed/given backbone 위에서 제한된 test-time compute 안의 heterogeneous reasoning path family를 구성·선택·검증·중단하는 **test-time path-pool allocation**으로 구분된다. STaR식 correctness-filtered rationale은 backbone을 얻는 한 방법일 뿐이며, 우리의 novelty는 그 위의 lightweight orchestration에 있다.
